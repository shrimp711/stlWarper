# stlWarper

stlWarper 是一个面向 FDM 3D 打印的 STL 预补偿网页工具。

它的核心目标只有一件事：在切片前先对模型做几何预变形，抵消打印冷却过程中的热收缩和翘曲，让成品更接近设计尺寸。

## stlWarper 是什么

stlWarper 是纯前端应用：

1. 上传 STL。
2. 选择材料和工艺温度预设。
3. 自动执行热变形预补偿。
4. 立即下载补偿后 STL。

所有计算都在本地浏览器执行，不依赖后端，不上传模型文件。

## 为什么 FDM 热变形补偿很重要

FDM 打印中，材料从熔融状态冷却到室温会产生非均匀收缩，常见后果包括：

- 翘边和翘角，底面难以贴合平台。
- 孔距和外形尺寸偏差，装配困难。
- 长薄结构弯曲，功能件失配。

尤其是 ABS 等高收缩材料，热应力更明显。仅靠切片参数和支撑并不总能解决问题。

stlWarper 通过材料-工艺对应的固有应变库，先把 STL 做“反向形变”，让打印后的自然收缩回到目标几何附近。

## 它有什么用

- 打印前快速预补偿，减少试错轮次。
- 用同一模型快速比较不同温度工艺下的补偿结果。
- 对功能件更友好，提升尺寸一致性与装配成功率。
- 在不改 slicer 和机器固件的前提下，增加一道几何级补偿能力。

## 1 分钟上手

### 1. 启动本地服务

请在项目根目录运行：

```powershell
Set-Location D:/Document/3Dprinting/Bumpmesh/stlWarper
python -m http.server 5173
```

然后在浏览器打开：

http://localhost:5173

注意：不要直接双击 index.html 用 file 协议打开，否则浏览器可能拦截 database.json 读取。

### 2. 在页面中操作

1. 上传 STL 文件。
2. 选择材料（如 PLA、ABS）。
3. 选择工艺预设（喷嘴/热床/腔体温度）。
4. 点击“一键补偿并下载”。

成功后会自动下载补偿模型，命名格式类似：

```text
原文件名_材料_喷嘴_热床_腔体_compensated.stl
```

## 当前实现原理（简版）

- 数据驱动：读取 material-db/database.json 中的固有应变参数。
- 几何代理：对模型顶点按高度和径向位置估计收缩/翘曲趋势。
- 迭代修正：多轮反向更新顶点坐标，直到误差收敛或达到迭代上限。
- 可视化核对：同屏显示原始模型与补偿后模型。

## 数据库说明

数据库路径：material-db/database.json

单条记录包含：

- material: 材料名
- t_melt: 喷嘴温度
- t_bed: 热床温度
- t_chamber: 腔体温度
- inherent_strain: 固有应变向量 [exx, eyy, ezz]
- status: 数据状态（success 才会被前端使用）

你可以按同样格式扩展更多材料和工艺点。

## 固有形变数据说明（请谨慎使用）

`inherent_strain` 当前主要来自仿真结果，用于快速建立预补偿能力。

请注意：仿真值与真实打印实验值可能存在偏差，偏差来源包括但不限于设备刚度、风道与散热条件、实际材料批次、切片策略和环境温度波动。

因此本工具中的固有形变参数应视为工程参考初值，不应直接等同于最终生产参数。建议在目标机型和目标材料上进行小样打印与实测标定，再迭代修正数据库后用于正式件。

## 项目结构

- index.html: 页面入口
- style.css: 页面样式
- js/app.js: UI 交互与流程控制
- js/compensator.js: 补偿计算和数据库索引
- js/viewer.js: Three.js 预览
- material-db/database.json: 补偿参数数据库

## 适用范围与边界

stlWarper 适合快速工程预补偿和工艺对比，不是高保真热-力耦合求解器。

对于以下场景，建议结合实测标定迭代使用：

- 大尺寸件、薄壁高翘曲件
- 多材料或复杂散热环境
- 对公差要求很严的精密装配件

## 常见问题

Q: 页面显示数据库加载失败？

A: 请确认通过本地静态服务访问（http://localhost:5173），不要用 file 协议直接打开。

Q: 为什么选项里看不到某些条目？

A: 只有 status 为 success 的记录会进入材料和工艺下拉列表。

Q: 这是离线的吗？

A: 模型计算在本地浏览器进行，不会上传 STL 到后端服务。
